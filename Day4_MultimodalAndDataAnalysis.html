课题8.5: 多模态处理三种模式对比

图片/视频/音频等非文本信息如何进入LLM？三种模式各有优劣

三种多模态处理模式

┌─────────┐ NATIVE ┌──────────┐ │ 图片 │ ──────────────► │ 多模态LLM │ → 直接理解 │ │ └──────────┘ │ │ EXTRACT ┌──────────┐ ┌──────────┐ │ │ ──────────────► │ OCR/检测 │→│ 文本LLM │ → 间接理解 │ │ └──────────┘ └──────────┘ │ │ TOOL ┌──────────┐ ┌──────────┐ │ │ ──────────────► │ 视觉工具 │→│ Agent+LLM│ → 结构化理解 └─────────┘ └──────────┘ └──────────┘
👁️
NATIVE（原生多模态）
图片直接送入多模态LLM（如GPT-4o），LLM直接理解视觉内容。保真度最高，但需要多模态模型，成本较高
📝
EXTRACT_TO_TEXT（提取为文本）
图片→OCR/目标检测→提取文本/标签→送入纯文本LLM。成本较低，但丢失空间关系和视觉语义
🔧
TOOL（多模态工具）
Agent调用视觉分析工具→获取结构化结果→送入LLM。灵活性高，可组合多个专用模型，但延迟较高
为什么需要三种模式？不能都用NATIVE吗？
理论上NATIVE模式最好——直接送入多模态LLM，信息无损失。但实际场景中有三个限制：
1. 成本：图片按Token计费（一张1024x1024图片约765 tokens），大量图片处理成本极高
2. 模型限制：不是所有LLM都支持多模态，纯文本LLM无法直接处理图片
3. 结构化需求：产线质检需要结构化输出（缺陷位置、类型、置信度），NATIVE输出是自然语言，需要额外解析

因此三种模式各有适用场景，实际部署中经常组合使用。

同一张PCB图片的三种模式处理对比

测试图片：一张PCB电路板产品照片，上面有多个电子元件、焊点和丝印标签

1

NATIVE模式 — 图片直接送入多模态LLM

输出包含：元件识别（U1-主控芯片、C1-C8贴片电容等）、缺陷检测（U1焊盘疑似桥接、C3偏移1mm）、空间关系（U1位于板中心，C1-C4环绕）、整体评估

✅ 优势: 保留完整视觉信息，可感知空间关系和外观细节

❌ 劣势: 需要多模态模型，token消耗大

2

EXTRACT_TO_TEXT模式 — OCR/检测提取后送文本LLM

OCR结果: 「PCB-V3.2 SN:2026070801 MADE IN CN」

检测标签: [芯片x1, 电容x8, 电阻x5, 连接器x1, 疑似桥接x1, 偏移x1]

关键缺失: 空间关系信息——无法从文本推断元件在板上的布局位置

✅ 优势: 成本低，可用纯文本LLM

❌ 劣势: 丢失空间关系、颜色、纹理等视觉信息

3

TOOL模式 — Agent调度专用视觉工具

工具调用链：Tool1: defect_detection(image) → {桥接:1, 偏移:1, 置信度:0.92}

Tool2: ocr_extract(image) → {丝印: 'PCB-V3.2', 序列号: '2026070801'}

Tool3: component_count(image) → {芯片:1, 电容:8, 电阻:5}

结构化输出：缺陷率2/15=13.3%，超出阈值5%，判定❌不合格，建议标记U1/C3区域生成返工工单

✅ 优势: 灵活组合专用模型，输出结构化，可执行后续动作

❌ 劣势: 多次调用延迟高，工具编排复杂度高

实战建议：组合使用三种模式
- NATIVE → 需要理解视觉细节：产品外观审核、设计稿评审、医学影像
- EXTRACT → 以文字信息为主：发票识别、证件录入、表格提取
- TOOL → 需要结构化输出+后续动作：质检流水线、安全巡检、自动告警

优先用TOOL模式（结构化+可执行），复杂视觉理解叠加NATIVE，OCR场景用EXTRACT。三种模式可组合使用！

课题9: 制造领域BI分析（含代码解释器）

从「描述性分析」升级到「推断性分析」——代码解释器让Agent能做统计推断

创建BI分析助手 + 绑定代码解释器

代码解释器：BI助手的关键升级
没有代码解释器：Agent只能说「良品率92.3%」——这是描述性分析，只是陈述事实
有了代码解释器：Agent可以说「p=0.003，温度与缺陷率显著正相关」——这是推断性分析，可以指导决策

代码解释器让Agent能够：
- 执行Python代码进行统计计算（标准差、相关系数、p值）
- 生成数据可视化图表
- 做假设检验和回归分析

关键配置：沙箱隔离(安全)、超时限制(30秒)、白名单库(pandas/numpy/scipy/matplotlib)
Python

课题10: 消费领域精准营销

RFM模型客户分群 + 差异化营销策略

RFM模型 + 客户画像

RFM模型：客户价值分群经典方法
RFM通过三个维度衡量客户价值：
- R(Recency)：最近消费时间 → 越近越活跃
- F(Frequency)：消费频次 → 越高越忠诚
- M(Monetary)：消费金额 → 越高越有价值

四类客户分群：
1. 高价值客户：R近、F高、M高 → VIP维护、专属权益
2. 成长客户：R近、F中、M中 → 提升频次、交叉推荐
3. 流失风险：R远、F低、M高 → 召回优惠、个性化推荐
4. 低价值客户：R远、F低、M低 → 促销引流、降低服务成本
