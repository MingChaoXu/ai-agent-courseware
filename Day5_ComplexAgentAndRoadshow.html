上午 · 课题15

社会治理综合智能体

4个Agent协同 + 异步处理(threading) + 条件分支 · 40分钟

核心目标

构建综治中心的多Agent协同系统，掌握异步并行处理（事件录入和态势预警同时执行）和条件分支（仅"矛盾纠纷"类型触发法律Agent）两个高级工作流模式。

⚡ 核心架构原理：异步+条件 = 高效+精准
异步处理：当两个Agent之间没有数据依赖时，可以并行执行。例如"事件录入"和"态势预警"——录入是处理当前事件，预警是分析月度趋势，两者互不依赖，并行可节省50%时间。

条件分支：不是每个事件都需要走所有Agent。矛盾纠纷类事件才需要法律咨询Agent，安全隐患类事件则直接进入简报生成。这通过工作流step中的condition字段实现："condition": "${standard_event.type} == '矛盾纠纷'"

异步处理架构

社会治理工作流（异步+条件分支）： 事件报告 ──► [Step1] 事件信息Agent（标准化录入） │ ┌─────────┴──────────┐ │ │ [Step2]法律咨询Agent [Step3]态势预警Agent 条件: type=="矛盾纠纷" 与Step2并行执行 │ │ └─────────┬──────────┘ │ [Step4] 简报生成Agent（汇总所有结果）

Python threading实现：使用threading.Thread和threading.Barrier协调两个并行线程。Barrier确保两个线程都完成后再进入简报汇总阶段。

2.2 条件分支详解

🔧 条件分支实现：工作流Step的condition字段
在工作流定义中，每个step可以添加condition字段，值为一个表达式。运行时，引擎会评估该表达式，仅当结果为True时执行该步骤。

示例："condition": "${standard_event.type} == '矛盾纠纷'"

这意味着：
- 上报漏水纠纷 → 事件Agent输出type="矛盾纠纷" → 法律Agent执行 → 简报包含法律建议
- 上报安全隐患 → 事件Agent输出type="安全隐患" → 法律Agent跳过 → 简报不含法律建议

条件分支让工作流"智能路由"，不同类型的事件走不同处理路径，避免无关Agent的无效调用。

========================================== # 步骤2: 编排异步处理 + 条件分支工作流 # ========================================== workflow = client.create_workflow( name="社会治理智能体（异步版）", description="事件录入→[并行]法律咨询+态势预警→简报生成", steps=[ { "step": 1, "name": "事件标准化录入", "agent_id": event_agent.get("id"), "input": "${event_report}", "output": "standard_event" }, { "step": 2, "name": "法律咨询（条件分支：仅纠纷类触发）", "agent_id": law_agent.get("id"), "condition": "${standard_event.type} == '矛盾纠纷'", "input": "${standard_event}", "output": "law_advice" }, { "step": 3, "name": "态势预警（与法律咨询并行）", "agent_id": alert_agent.get("id"), "input": "${monthly_data}", "output": "alert_result", "parallel_with": 2 }, { "step": 4, "name": "简报生成（汇总所有结果）", "agent_id": brief_agent.get("id"), "input": { "event": "${standard_event}", "law": "${law_advice}", "alert": "${alert_result}", "monthly": "${monthly_data}" }, "output": "brief_report" } ] ) # 并行执行事件录入和态势预警 results = {} barrier = threading.Barrier(2, timeout=120) def _thread_event(): results["event"] = _run_event_entry(event_report) barrier.wait() def _thread_alert(): results["alert"] = _run_alert_check(monthly_data) barrier.wait() t1 = threading.Thread(target=_thread_event) t2 = threading.Thread(target=_thread_alert) start_time = time.time() t1.start() t2.start() t1.join(timeout=120) t2.join(timeout=120) elapsed = time.time() - start_time print(f"\n 并行执行耗时: {elapsed:.1f}s") print(" （串行预估耗时: 2x 以上）") # ========================================== # 步骤4: 条件分支测试 # ========================================== # 测试1: 矛盾纠纷类 → 触发法律Agent result_law = client.chat(law_agent.get("id"), "这是一起邻里漏水纠纷：楼上装修导致楼下天花板损坏，" "三方（楼上业主、装修队、楼下业主）责任推诿。请提供法律建议。") print(f"🤖 法律咨询:\n{result_law.get('content', '')}") # 测试2: 安全隐患类 → 不触发法律Agent safety_report = "网格员上报：XX路施工现场围栏破损，行人可直接进入危险区域。" result_safety = client.chat(event_agent.get("id"), f"请将以下事件标准化录入：\n{safety_report}") print(f"🤖 事件录入（无法律分支）:\n{result_safety.get('content', '')}") # 简报汇总 result_brief = client.chat(brief_agent.get("id"), f"请根据以下信息生成治理简报：\n" f"月度数据：{monthly_data}\n" f"新增纠纷事件：邻里漏水纠纷") print(f"🤖 治理简报:\n{result_brief.get('content', '')}") print("\n✅ 课题15完成！")
下午 · 课题16

智能体消融评估实战

消融实验 + 量化分析 + 评估报告 · 50分钟

核心目标

掌握消融评估方法——通过系统化移除Agent组件来量化各部分贡献度，回答"到底哪个组件最重要"这个关键问题。

⚡ 核心架构原理：消融实验 = 科学验证组件价值
消融实验（Ablation Study）源自神经科学研究：移除大脑某区域，观察功能变化，从而推断该区域的功能。

在Agent评估中，消融实验的方法：
1. 建立基线（baseline）：完整配置的Agent
2. 逐个移除组件：移除system_prompt、移除工具描述、改非正式语气...
3. 在相同测试集上评估每个变体
4. 计算各组件的贡献度 = |变体得分 - 基线得分| / 基线得分 × 100%

关键公式：contribution_pct = |delta| / baseline_score * 100%
影响等级：🔴 重大 >30% 🟡 显著 >15% 🟢 轻微 ≤15%

4.1 建立评估基准

5个测试问题

  1. 公积金购房提取需要哪些材料？
  2. 户口从外地迁入苏州怎么办理？
  3. 社保断缴3个月会有什么影响？
  4. 不动产登记需要多长时间？
  5. 婚姻登记可以跨区办理吗？

4维度评估函数

🔧 evaluate_answer(question, answer) → 0.0~1.0
维度1：完整性（0.3分）
回答长度 > 50字 → +0.2，> 150字 → +0.1
衡量Agent是否给出了充分的信息

维度2：准确性（0.3分）
根据问题关键词匹配答案中的专业术语。例如"公积金"问题检查["提取","账户","余额","购房"]的命中率
命中率 = 匹配词数 / 总关键词数 × 0.3

维度3：结构性（0.2分）
检查回答中是否有结构化标记：编号(1. 2.)、步骤(① ②)、列表(- )
有任意一个 → +0.2

维度4：政策引用（0.2分）
检查回答中是否有政策引用标记：规定、政策、条例、办法、通知
有任意一个 → +0.2

4.2 消融实验4种配置

配置名称描述移除/修改的组件
baseline完整配置无（基准线）
no_system_prompt移除System Prompt清空system_prompt，Agent不知道自己是谁
no_tool_description移除工具描述简化职责描述和输出格式要求
informal_tone改为非正式语气将专业提示词替换为随意口吻
💡 预期实验结果
no_system_prompt → 影响最大 🔴
移除后Agent不知道自己是政务助手，可能回答生活常识而非政务指南，准确率可能下降30%以上

no_tool_description → 影响显著 🟡
简化职责描述后，Agent可能遗漏关键信息（如不引用政策依据），结构性评分下降

informal_tone → 影响轻微 🟢
语气变化不影响信息准确性，但可能影响用户信任度和专业感知

4.3 量化分析与影响等级

消融实验的量化分析核心是贡献度排序：

System Prompt
~35% 🔴 重大
工具描述
~22% 🟡 显著
语气风格
~10% 🟢 轻微
💡 量化洞察
1. System Prompt是影响最大的组件 → 移除后准确率可能下降30%+，是Agent的"灵魂"
2. 工具描述决定了LLM能否正确选工具 → 移除后工具调用失败率飙升
3. 语气风格影响专业性感知 → 评分下降但功能不受损，可按场景灵活调整
4. 建议每次迭代都跑消融实验 → 验证新改动确实带来了正向效果

评估报告4章节

  1. 消融实验结果：各配置的平均分和变化量
  2. 各问题详细得分：每个问题在各配置下的得分
  3. 核心结论：按影响程度排序的组件贡献度
  4. 优化建议：优先保障System Prompt质量、工具描述需精准、语气可灵活调整

""" 课题16: 智能体消融评估实战 消融实验 + 量化分析 + 评估报告 """ import sys, os, json, time sys.path.insert(0, os.path.join(os.path.dirname(__file__), "..", "common")) from teleagent_client import TeleAgentClient, get_client from training_utils import AblationFramework def exercise16_run(): client = get_client() # ========================================== # 步骤1: 用AblationFramework建立评估基准 # ========================================== baseline_agent = client.create_agent( name="评估基准Agent", description="消融实验基准智能体", system_prompt=( "你是政务问答助手。\n" "职责：回答户籍、社保、公积金、不动产等政务咨询。\n" "要求：回答准确、条理清晰、引用政策依据。\n" "输出格式：结论 + 政策依据 + 办理指引" ), model="default" ) agent_id = baseline_agent.get("id") # 定义测试问题集 test_questions = [ "公积金购房提取需要哪些材料？", "户口从外地迁入苏州怎么办理？", "社保断缴3个月会有什么影响？", "不动产登记需要多长时间？", "婚姻登记可以跨区办理吗？", ] # 定义评估函数（4维度） def evaluate_answer(question: str, answer: str) -> float: """基于多维度评分的评估函数""" if not answer or len(answer) < 10: return 0.0 score = 0.0 # 维度1: 完整性 if len(answer) > 50: score += 0.2 if len(answer) > 150: score += 0.1 # 维度2: 准确性 accuracy_keywords = { "公积金": ["提取", "账户", "余额", "购房"], "户口": ["迁入", "迁出", "派出所", "户籍"], "社保": ["断缴", "补缴", "缴费", "养老"], "不动产": ["登记", "房产", "证书", "办理"], "婚姻": ["登记", "民政局", "跨区", "预约"], } for q_key, keywords in accuracy_keywords.items(): if q_key in question: matched = sum(1 for kw in keywords if kw in answer) score += 0.3 * (matched / len(keywords)) break # 维度3: 结构性 structural_markers = ["1.", "2.", "①", "②", "第一步", "- "] if any(m in answer for m in structural_markers): score += 0.2 # 维度4: 政策引用 policy_markers = ["规定", "政策", "条例", "办法", "通知"] if any(m in answer for m in policy_markers): score += 0.2 return min(1.0, score) # ========================================== # 步骤2: 消融实验 - 4种配置 # ========================================== ablation_variants = { "baseline": { "description": "完整配置（system_prompt + 工具描述 + 专业语气）", "system_prompt": ( "你是政务问答助手。\n" "职责：回答户籍、社保、公积金、不动产等政务咨询。\n" "要求：回答准确、条理清晰、引用政策依据。\n" "输出格式：结论 + 政策依据 + 办理指引" ) }, "no_system_prompt": { "description": "移除System Prompt", "system_prompt": "" }, "no_tool_description": { "description": "移除工具描述", "system_prompt": ( "你是政务问答助手。\n" "职责：回答政务咨询。\n" "输出格式：结论 + 办理指引" ) }, "informal_tone": { "description": "改为非正式语气", "system_prompt": ( "你是个热心的政务咨询小伙伴~\n" "帮大家回答政务问题，随意一点就好，不用太正式。\n" "知道啥说啥就行~" ) }, } def agent_fn_baseline(question: str) -> str: result = client.chat(agent_id, question) return result.get("content", "") framework = AblationFramework( agent_fn=agent_fn_baseline, evaluator_fn=evaluate_answer ) # 运行消融实验 all_results = [] mode_labels = { "baseline": "完整配置", "no_system_prompt": "移除System Prompt", "no_tool_description": "移除工具描述", "informal_tone": "非正式语气" } for mode, config in ablation_variants.items(): print(f"\n🔬 消融模式: {mode} - {config['description']}") mode_scores = [] if mode == "baseline": for q in test_questions: answer = agent_fn_baseline(q) score = evaluate_answer(q, answer) mode_scores.append(score) else: variant_agent = client.create_agent( name=f"消融变体-{mode}", description=config["description"], system_prompt=config["system_prompt"], model="default" ) for q in test_questions: result = client.chat(variant_agent.get("id"), q) score = evaluate_answer(q, result.get("content", "")) mode_scores.append(score) avg_score = sum(mode_scores) / len(mode_scores) if mode_scores else 0 all_results.append({ "mode": mode, "label": mode_labels[mode], "avg_score": round(avg_score, 3), "scores": [round(s, 3) for s in mode_scores] }) # ========================================== # 步骤3: 量化分析各组件贡献度 # ========================================== baseline_score = all_results[0]["avg_score"] for r in all_results: r["delta"] = round(r["avg_score"] - baseline_score, 3) r["contribution_pct"] = round( abs(r["delta"]) / baseline_score * 100, 1 ) if baseline_score > 0 else 0 print(f"{'消融模式':<22} {'平均分':>8} {'变化量':>8} {'贡献占比':>8} {'影响等级':>10}") print("-" * 62) for r in all_results: delta_str = f"{'+' if r['delta'] >= 0 else ''}{r['delta']}" if r["mode"] == "baseline": impact = "— (基准)" elif abs(r["contribution_pct"]) > 30: impact = "🔴 重大" elif abs(r["contribution_pct"]) > 15: impact = "🟡 显著" else: impact = "🟢 轻微" print(f"{r['label']:<22} {r['avg_score']:>8.3f} {delta_str:>8} " f"{r['contribution_pct']:>7.1f}% {impact:>10}") # ========================================== # 步骤4: 生成评估报告（4章节） # ========================================== report_lines = [] report_lines.append("=" * 60) report_lines.append("智能体消融评估报告") report_lines.append("=" * 60) report_lines.append(f"评估时间: {time.strftime('%Y-%m-%d %H:%M:%S')}") report_lines.append("被测智能体: 政务问答助手") report_lines.append(f"测试问题数: {len(test_questions)}") report_lines.append("评估维度: 完整性 + 准确性 + 结构性 + 政策引用") report_lines.append("") report_lines.append("一、消融实验结果") report_lines.append("-" * 40) for r in all_results: delta_str = f"{'+' if r['delta'] >= 0 else ''}{r['delta']}" report_lines.append(f" {r['label']}: {r['avg_score']:.3f} (变化: {delta_str})") report_lines.append("") report_lines.append("二、各问题详细得分") report_lines.append("-" * 40) for i, q in enumerate(test_questions): report_lines.append(f" Q{i+1}: {q[:30]}...") for r in all_results: report_lines.append(f" {r['label']}: {r['scores'][i]:.3f}") report_lines.append("") report_lines.append("三、核心结论") report_lines.append("-" * 40) sorted_results = sorted(all_results[1:], key=lambda x: abs(x["delta"]), reverse=True) for i, r in enumerate(sorted_results): report_lines.append(f" {i+1}. {r['label']}影响最大，贡献占比{r['contribution_pct']}%") report_lines.append("") report_lines.append("四、优化建议") report_lines.append("-" * 40) report_lines.append(" 1. 优先保障System Prompt的质量和完整性") report_lines.append(" 2. 工具描述需精准，避免LLM选错工具") report_lines.append(" 3. 语气风格可按业务场景灵活调整") report_lines.append(" 4. 建议每次迭代都跑消融实验验证效果") report_text = "\n".join(report_lines) print(report_text) # 综合方案路演准备

路演四大板块框架 + TeleAgent完整Demo搭建 + 评分体系 + Checklist · 70分钟

核心目标

将5天所学整合为一次完整路演——从场景分析到技术方案到Demo演示到商业价值，形成端到端的方案展示能力。

⚡ 路演核心公式：场景 × 技术 × Demo × 价值 = 说服力
路演不是技术报告，而是说服决策者投资的过程。四大板块各有侧重：

1. 场景分析（20%）：让听众感受到"这个问题确实需要解决"
2. 技术方案（30%）：让听众相信"这个方案能解决问题"
3. Demo演示（30%）：让听众看到"方案已经在跑了"
4. 商业价值（20%）：让听众确信"投入产出比值得"

注意权重分配：技术+Demo占60%，因为决策者最关心"能不能做"和"做出来长什么样"。

5.1 路演四大板块框架

场景分析
权重 20%
· 业务痛点描述 (5分)

商业价值
权重 20%

· 推广路径可行 (5分)
· 商业模式清晰 (5分)
· 二期规划远见 (4分)
🔧 每个板块的关键策略
场景分析关键：用量化数据支撑痛点。不要说"效率低"，要说"平均每个工单处理耗时45分钟，其中60%是重复性填表操作"。

技术方案关键：架构图要清晰展示Agent拓扑+数据流+事件总线。创新点要与通用方案区分——"我们用了事件驱动+System Hint"比"我们用了AI"更有说服力。

Demo演示关键：端到端流程必须跑通，提前录制备份视频。展示事件触发（不是手动调API），展示异常处理（不是只演示Happy Path）。

商业价值关键：ROI要可信——"人效提升40%"比"大幅提升"有说服力。推广路径分3阶段：试点→区县→全市。
