## 课题名称

Prompt 消融实验

## 学习目标

- 掌握科学评估 Prompt 各组件贡献的方法
- 理解消融实验设计原理

## 任务要求

1. **选择基础 Prompt**：
   - 以合同审核（ex4）的 Prompt 为基础
   - 完整版 Prompt 应包含：角色定义、输出格式要求、示例（few-shot）、约束条件

2. **逐步消融组件**：
   - **实验1**：移除角色定义，保留其余组件
   - **实验2**：移除输出格式要求，保留其余组件
   - **实验3**：移除示例（few-shot），保留其余组件
   - **实验4**：移除约束条件，保留其余组件
   - **实验5**：仅保留核心指令（移除全部辅助组件）

3. **量化评估**：
   - 对每次消融后的输出进行评分，维度包括：格式正确性、内容完整性、风险识别准确率、输出一致性
   - 每个维度 1-5 分，计算总分变化
   - 在相同输入上重复 3 次取平均，减少随机性影响

4. **生成实验报告**：
   - 汇总为消融实验表格：移除哪项 → 各维度分数变化 → 总分影响
   - 按影响程度对组件排名
   - 得出结论：哪些组件是关键组件

## 技术栈

- LangChain 1.x（ChatPromptTemplate, StrOutputParser, LCEL）

## 输入数据

- 同 ex4 的合同样本数据

## 预期输出

- 消融实验表格（移除哪项 → 质量影响多大）
- 关键组件排名

## 提示与思考

- 消融实验的变量控制：除了被消融的组件，其他条件应严格一致。你在实验中如何保证？
- 组件之间是否存在交互效应？即：同时移除 A 和 B 的影响，可能不等于分别移除 A 和 B 的影响之和。你会如何验证？
- 3 次重复是否足够？如果某次结果偏差较大，应该如何处理？
- 消融实验的结论是否可迁移？在合同审核场景得出的组件排名，是否适用于其他任务？
