## 课题名称

工程建设文档生成

## 学习目标

- 掌握 ChatPromptTemplate 的动态变量用法
- 实现一套 Prompt 适配多种公文文种

## 任务要求

1. **多文种支持**：
   - 用 `doc_type` 变量控制文种类型，支持：施工方案、监理规划、安全预案、进度报告
   - 不同文种对应不同的 System Prompt（角色定位、写作风格、格式要求）

2. **动态 Prompt 切换**：
   - 设计 Prompt 模板，根据 `doc_type` 参数自动选择对应的 System Prompt
   - 共享输入变量：项目名称、工期、规模等工程基本信息
   - 不同文种的输出格式要求不同（如施工方案需分章节，进度报告需时间轴）

3. **自由格式输出**：
   - 使用 StrOutputParser 输出纯文本公文
   - 各文种的文档结构应与真实公文格式一致

## 技术栈

- LangChain 1.x（ChatPromptTemplate, StrOutputParser, LCEL）

## 输入数据

- 工程基本信息（项目名称、工期、规模、参建单位等）
- 文种类型（doc_type 参数）

## 预期输出

- 4 种不同文种的工程建设文档

## 提示与思考

- 如何设计模板让 4 种文种共享通用逻辑又各有特色？试试用条件分支或字典映射管理多套 System Prompt。
- 真实公文有严格的格式规范（如章节编号、表格样式），纯 LLM 生成的文本能否满足？如果需要更精确的格式控制，你会怎么改进？
- 施工方案和安全预案在内容上有大量重叠，如何避免重复生成？
- 如果要支持用户自定义文种（如"竣工验收报告"），你的模板设计是否具备扩展性？
