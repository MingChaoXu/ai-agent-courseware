## 课题名称

低空智能体协同管控平台

## 学习目标

- 理解多 Agent 编排架构：Dispatcher 路由模式如何将用户指令分发到不同专业 Agent
- 掌握 CV 视觉模型（YOLOv8）的部署与 API 封装，理解大模型 Agent 如何调用视觉工具
- 理解"文本对话 + 图像检测 + 地图可视化"的端到端多模态融合闭环

## 项目背景

低空经济正在快速发展，城市低空空域的管理需要协调感知、物流、管制、应急等多个专业领域。你将体验一个低空智能体协同管控平台：四大智能体各司其职，共享空域数据和地图可视化，并能按需调用 YOLOv8 视觉检测服务分析航拍图像。通过自然语言对话驱动整个低空生态运行，理解多 Agent + CV + 地图的融合架构。

## 功能需求

### 全域感知智能体

负责空域态势监控：报告当前无人机编队状态、检测异常飞行目标（鸟类、未授权飞行器等）、识别空域障碍物、提供态势摘要报告。

### 无人机物流智能体

负责物流订单调度：处理配送订单、规划航线、分配无人机、评估降落点安全性（调用视觉工具检测降落区域人员和障碍物密度）。

### 城市空中交通智能体

负责空域交通管控：分配飞行空域、检测航线冲突、管控交通流量、分析地面交通态势（调用视觉工具检测车流密度）。

### 应急响应智能体

负责突发情况处置：低电量迫降、未授权入侵、自然灾害侦察等应急场景的分级响应，调用视觉工具评估灾害现场状况。

### CV 视觉检测服务

基于 YOLOv8 的独立检测服务，支持多种检测模式：航拍全量检测、障碍物识别、入侵检测、降落点评估、灾害分析、交通监控。各 Agent 可按需调用不同检测模式。

### 地图可视化

在地图上实时展示无人机位置、航线、检测事件等信息，支持缩放和交互操作。

## 实验任务

### 基础：启动与验证

1. 配置环境（LLM API Key + YOLOv8 模型权重）
2. 启动所有服务，确认 LLM 状态和 CV 模型状态正常
3. 验证地图显示无人机编队初始位置

### 感知体验：空域态势感知

1. 输入"报告当前空域态势"，观察 Dispatcher 路由到感知 Agent
2. 输入"调用 CV 检测分析航拍画面中的目标"，观察 CV 检测结果面板

### 物流体验：订单调度

1. 输入物流配送需求，观察物流 Agent 的航线规划和调度
2. 输入"评估降落点安全性"，观察 Agent 自动调用 landing 模式检测

### 管制体验：交通管控

1. 输入航线冲突检测请求，观察管制 Agent 的冲突分析
2. 输入交通态势分析请求，观察 Agent 调用 traffic 模式检测

### 应急体验：应急处置

1. 输入低电量迫降指令，观察应急 Agent 的分级响应
2. 输入灾害侦察请求，观察 Agent 调用 disaster 模式检测

## 技术要求

- 使用 Dispatcher 路由模式将用户输入分发到对应 Agent
- CV 服务独立部署，通过 HTTP API 供 Agent 调用
- 各 Agent 可根据需要自主决定是否调用 CV 检测，调用哪种检测模式
- 地图使用高德地图 API 或 Leaflet + OpenStreetMap
- 后端使用 FastAPI，前端使用 Vue 3（CDN 引入）

## 预期效果

- 自然语言对话即可驱动四大 Agent 完成空域管理全流程
- CV 检测结果和标注图片在面板中实时展示
- 地图实时显示无人机位置和状态
- Dispatcher 路由决策可视化，可观察指令分发过程

## 提示与思考

- Dispatcher 的路由策略如何设计？如何处理跨 Agent 的复合问题？
- CV 检测结果如何更好反馈给 Agent 做更精准决策？结构化 vs 自然语言描述哪个更好？
- 如果要支持实时视频流检测（而非单帧图片），架构需要怎么调整？
- 多 Agent 之间的状态共享通过数据库实现，有什么优缺点？还有其他方案吗？
