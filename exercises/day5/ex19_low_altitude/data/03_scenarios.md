# 模拟应急场景

## 场景1 - 无人机航线偏离
- 事件类型：route_deviation
- 涉及无人机：UAV-003 信鸽
- 描述：UAV-003在执行订单#002配送途中，偏离预定航线约200米，当前位置偏西南方向
- 严重程度：警告
- 位置：30.5628, 104.0568
- 可能原因：侧风影响 / GPS信号弱 / 飞控异常
- 建议处置：
  1. 感知Agent：持续监控偏离趋势，评估是否继续扩大
  2. 管制Agent：检查该空域是否有其他飞行器，评估冲突风险
  3. 物流Agent：评估是否需要调整航线或更换无人机
  4. 应急Agent：若偏离超过500米，触发返航程序

## 场景2 - 恶劣天气预警
- 事件类型：weather_alert
- 描述：气象系统检测到龙泉驿方向侧风风速12m/s，阵风可达15m/s
- 严重程度：警告
- 位置：30.5828, 104.0768
- 影响范围：龙泉驿方向航线，高度100m以下
- 建议处置：
  1. 在飞无人机降低速度，提升高度至150m以上
  2. 待飞订单延迟起飞
  3. 已飞往该区域的无人机评估备降方案
  4. CV视觉服务：可调用航拍检测，辅助判断地面天气状况

## 场景3 - 无人机低电量
- 事件类型：low_battery
- 涉及无人机：UAV-004 翼龙
- 描述：UAV-004电量低于30%，当前在充电状态但充电缓慢
- 严重程度：危险
- 位置：30.5728, 104.0668
- 建议处置：
  1. 应急Agent：评估UAV-004当前任务是否可转移
  2. 物流Agent：将待分配订单转给其他可用无人机
  3. 持续监控充电进度，电量恢复至80%方可重新执行任务

## 场景4 - 空域入侵（模拟）
- 事件类型：airspace_intrusion
- 描述：检测到未授权飞行器进入核心管控区，疑似消费级无人机或大型鸟类
- 严重程度：危险
- 位置：30.5778, 104.0718
- 建议处置：
  1. 感知Agent：调用CV检测(intruder模式)识别目标类型
  2. 管制Agent：周边无人机紧急避让，建立安全隔离区
  3. 应急Agent：评估入侵目标威胁等级
  4. 若为无人机：上报管控中心，尝试信号干扰
  5. 若为鸟类：调整航线避开鸟群活动区域

## 场景5 - 灾害侦察（模拟）
- 事件类型：disaster_recon
- 描述：都江堰方向报告疑似山体滑坡，需要紧急派遣无人机进行航拍侦察
- 严重程度：危险
- 位置：30.9828, 103.6268
- 建议处置：
  1. 感知Agent：调度UAV-004前往事发区域（如电量允许）
  2. CV检测：到达后调用disaster模式进行灾害场景分析
  3. 应急Agent：根据CV检测结果评估灾害范围和严重程度
  4. 物流Agent：评估是否可同时配送应急物资
  5. 管制Agent：开辟应急通道，保障救援无人机优先通行

## 场景6 - 降落点安全评估（模拟）
- 事件类型：landing_assessment
- 描述：UAV-002执行急救药品配送即将到达龙泉驿站，需要评估降落点安全性
- 严重程度：提示
- 位置：30.6128, 104.1068
- 建议处置：
  1. CV检测：调用landing模式分析降落区域图像
  2. 评估项：地面平整度、障碍物、人员密度、车辆情况
  3. 若检测结果为danger：选择备用降落点（LP-04西侧50m）
  4. 若检测结果为normal：正常执行降落程序
